# MEMORY.md

## 核心任務：區塊鏈幣流分析教學軟體
- **定位**：繁體中文版的單機教學與分析軟體，非全自動黑箱作業，而是引導使用者學習如何透過公開帳本追查幣流。
- **核心功能**：完整紀錄操作流程（足跡追蹤），產出符合司法機關程序標準的分析報告。
- **報告格式**：支援 `.doc`, `.odt`, `.csv`, `.pdf`, `.png`。
- **法庭展演功能 (Courtroom Replay)**：具備如同「迷宮破解」般的視覺化互動展示能力，能在法庭上直接執行，一步步重現資金追查的思考邏輯與操作過程。核心目標為解決「如何將複雜交易流程與巨量資料轉化為法官/陪審團可理解、可驗證之證據鏈」的司法實務痛點。
- **Lumen 的角色**：頭號代理人、專案主管兼資安長 (CISO)。負責規劃團隊分工與監督專案安全性。

## 專案限制與戰略考量 (2026-03-25 更新)
- **開發資源**：Deede 是獨立開發者 (Solo Developer)，目前零預算、無外部研究資源。
- **資安與隱私 (偵查不公開)**：完全單機運行。架構採「自備金鑰 (Bring Your Own Key, BYOK)」，由軟體教學並引導調查員自行申請免費 API (如 Etherscan, Alchemy)，確保查詢紀錄不與軟體開發者綁定。
- **成本控管**：Lumen 負責監控開發與維運成本，必須設計一套零成本/低成本的技術堆疊。
- **終極目標**：本專案的成果與過程將寫成**研究論文 (Research Paper)**。

## 關鍵決策與 UI 設計理念 (2026-03-26 更新)
- **主動式防呆與驗證機制**：系統不允許使用者無腦操作。系統底層透過 API 抓取的數據預設隱藏，會強制引導使用者自行前往官方瀏覽器查詢、下載 CSV 並匯入，最後由系統進行自動比對驗證。確保使用者真正學會操作並核對資料完整性（有無缺漏/授權問題）。
- ** MistTrack 整合教學**：在初勘步驟加入引導使用者至 MistTrack 查詢標籤的教學任務。
- **區分申請與啟用時間**：明確區分錢包的「創建(申請)時間」與「第一次入金(啟用)時間」，並強制列出提供最多 TRX/Gas 的供應者。
- **證據鏈開端 (身分登錄)**：軟體開啟後強制進行「調查員身分登錄（單位/職稱/姓名）」，並綁定操作時間，常駐於畫面右上角，確保證據不可否認性。
- **雙軌匯率驗證機制**：介面上教導調查員前往 CMC 查詢當日最高/最低價平均值並填入；系統底層則偷偷呼叫 Binance/CoinGecko 的「免費公開 API」進行交叉比對，若誤差 > 5% 則強制阻擋並糾正，達成零成本防呆。

# UI Wireframe Script: 初勘室 (Triage) v1.1 (Deede 修正版)

## 1. 介面總覽與設計理念

**系統運作邏輯（防呆與教學並重）**：
系統不會讓使用者無腦操作。當使用者輸入地址後，系統背景透過 API 查到的資料會先隱藏（存放在 `data` 中）。系統會先跳出「教學任務」，引導使用者親自前往原生帳本瀏覽器（如 Tronscan / Etherscan）查詢並下載交易紀錄（CSV），將其匯入軟體後，系統才會自動比對使用者匯入的紀錄與系統底層抓取的紀錄是否有誤/缺漏。

**設計理念（Lumen 的偵查思維）**：「拿到地址的第一秒，不是去搜，而是先分類。」
第一步是先教會使用者判斷：所拿到的錢包是否正確、有無缺漏、判斷鏈別、以及有無被惡意合約授權。此介面的核心目標是：在展開複雜的遞迴資金流向圖之前，引導調查員先對該地址進行快速的體檢與分類，建立證據足跡的基石。

---

## 2. 介面佈局 (Layout Structure)

整個初勘室分為「三大區塊」：頂部資訊列、左側互動教學檢查表、右側關鍵數據輔助區。

### 區塊一：頂部資訊列 (Header / Quick Stats)

提供目標地址的基礎公開帳本資料，快速判別該地址的活躍程度。
- **目標地址**: 顯示完整地址，並附帶 `[一鍵複製]` 與 `[QR Code]` 按鈕。
- **鏈別標籤 (Chain Tag)**: 系統自動辨識並高亮顯示（例：`TRC-20` (Tron) 或 `ERC-20` (EVM)）。
- **生命週期**: `[申請時間 YYYY-MM-DD]` / `[首次啟用(入金)時間 YYYY-MM-DD]`。
- **交易活躍度**: 總交易次數 `[數字]` 筆。
- **當前餘額**:
  - 主鏈原生代幣 (如 TRX / ETH)
  - 核心穩定幣/代幣 (如 USDT)

---

### 區塊二：左側 - 互動教學檢查表 (Interactive Triage Checklist)

這是初勘室的**核心功能**。結合 Lumen 導師的教學引導，要求調查員逐步完成三項關鍵判定。此區域採「卡片式手動勾選」設計，每一次勾選都會寫入本機端的「足跡引擎」資料庫中，作為最終產出鑑識報告的依據。

#### 🕵️‍♂️ 檢查點 A：啟動資金來源 (Genesis & Gas Source)

- **Lumen 導師氣泡提示**: 
  - 「這個錢包活過來的錢是誰給的？手續費是自己付的，還是別人贊助的？」
  - 「請注意：錢包被『創建(申請)』的時間，不一定等於它『第一次有錢進來(啟用)』的時間。我們要抓的是第一筆轉入的 TRX/ETH。」
  - 「👉 教學任務：請前往 MistTrack (https://misttrack.io/) 查詢此來源地址是否有被標記為高風險或已知實體。」
- **UI 呈現**:
  - 顯示該地址歷史上的「第一筆原生代幣轉入紀錄」(包含 TxID, 日期, 金額)。
  - `From: [來源地址]` ➔ `To: [目標地址]`
  - 特別標註：**「查明誰提供最多的 TRX/Gas」**（列出累計提供最多手續費的 Top 1 地址）。
- **互動判定 (請勾選最符合的特徵)**:
  - [ ] **來自大型交易所**: 來源地址帶有交易所標籤。
  - [ ] **來自個人/未標記地址**: 警示！該來源極可能是「資源供應源 (Gas Provider)」，建議加入關聯集群 (Cluster) 追蹤。
  - [ ] **涉及資源租賃**: (Tron 專屬) 發現交易帶有極小額 TRX 或代理能源合約互動，為典型的規避追蹤手法。

#### 🕵️‍♂️ 檢查點 B：權限是否轉移 (Permission & Control)

- **Lumen 導師氣泡提示**: 「這是一個獨立錢包，還是被人完全控制的傀儡？」
- **UI 呈現**:
  - 呼叫合約或鏈上權限 API，顯示當前地址的控制權 (Owner/Active Permission)。
- **互動判定 (請勾選驗證結果)**:
  - [ ] **權限未轉移**: 由原建立者 100% 掌控。
  - [ ] **權限已轉移 (多簽/變更)**: 警示！控制權已移交至 `[另一地址]`，判定為「傀儡錢包」，犯嫌可能隨時清空資產。

#### 🕵️‍♂️ 檢查點 C：是否為交易所水庫 (Hot Wallet / Deposit)

- **Lumen 導師氣泡提示**: 「觀察它的行為特徵。它是散戶、過水入金地址，還是交易所的大水庫？」
- **UI 呈現**:
  - **[手動展開交易紀錄]**: 交易紀錄預設隱藏，需由調查員主動點擊呼叫才會顯示近期流向圖表（例如「多進單出」或「單進單出」）。
  - 標籤庫比對結果（例如：`Binance-Hot`）。
- **互動判定 (請依特徵進行分類標記)**:
  - [ ] **交易所水庫 (Hot Wallet)**: 交易頻繁、總量極大，且公開標籤顯示為已知交易所熱錢包。
  - [ ] **入金地址 (Deposit Address)**: 資金匯入後，隨即有「歸集」動作，將款項全數轉至某個大水庫。
  - [ ] **一般涉案地址**: 無明顯上述特徵，通常為中繼站、找零地址或個人錢包。

---

### 區塊三：右側 - 關鍵交易雷達 (Key Transactions Radar)

在調查員操作左側檢查表時，右側提供數據輔助，讓調查員無需跳出畫面即可確認細節。

- **匯入/驗證狀態區**: 顯示調查員手動匯入的 CSV 紀錄與系統底層數據的「比對狀態（吻合/有缺漏）」。
- **最新異常大額轉帳**: 過濾掉粉塵攻擊 (Dusting)，僅列出前 5 筆大額進出。
- **合約授權 (Approve) 預警區**: 列出該地址曾授權 USDT 等代幣轉移權限的可疑智能合約（偵測授權詐騙/釣魚）。
- **資源消耗圖表 (Tron 特化)**: 以長條圖或圓餅圖顯示近期交易是否有高頻率的能量 (Energy) 租借行為。

---

## 3. 底部決策列與背景行為 (Action Bar & Background Engine)

當使用者完成左側三項檢查表的互動判定，且 **匯入的交易紀錄驗證無誤** 後，底部按鈕解鎖：

1. **[生成初勘紀錄節點]**: 將剛剛勾選的分類結果、擷取的原始 JSON 資料與時間戳記，雜湊加密後存入本機 SQLite。
2. **[進入資金追蹤迷宮 (Tracing Map)]**: 正式進入下一階段，開啟互動式資金追蹤介面，將此地址設定為起點 (Root Node)。

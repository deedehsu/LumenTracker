# UI Wireframe Script: 介面一與介面二 (證據鏈與案情登錄)

## 介面一：調查員身分登錄 (證據鏈起點)
**目的：** 確立數位證據的不可否認性。
- 強制輸入：`所屬單位`、`職稱/職別`、`姓名`。
- 背景行為：抓取並鎖定當前系統精確時間 (YYYY-MM-DD HH:MM:SS) 作為專案「建檔時間」。
- UI 回饋：登入後，上述資訊常駐於軟體右上角 (Header) 隨時可見。

## 介面二：案件基本資料設定
**目的：** 將紙本報案資料 (如筆錄、匯款紀錄) 數位化並建立追查基準點。

### 區塊 A：案情摘要
- 文字方塊：供調查員輸入報案重點 (將作為最終報告前言)。

### 區塊 B：登記已知涉案錢包地址
- 輸入框：錢包地址。
- 下拉選單：標籤設定 (`被害人`, `幣商`, `入金錢包`, `出金錢包`, `詐團水庫`)。
- 行為：可新增多筆，並提供移除功能。

### 區塊 C：新增已知交易紀錄與「當日匯率」驗證 (重點防呆)
- **輸入欄位：**
  - 交易日期 (`YYYY-MM-DD`)
  - 交易時間 (`HH:MM`)
  - 來源地址 (`From`)
  - 去向地址 (`To`)
  - 金額 (`新臺幣 TWD`)
  - 幣種 (`如 USDT`)
  - 數量 (`虛擬資產數量`)
  - **計算當日匯率** (`使用者手動輸入`)
- **藍色教學面板：** 引導使用者前往 CoinMarketCap 查詢當日該幣種的「最高價」與「最低價」，並教導公式：`(最高價 + 最低價) ÷ 2 = 當日匯率`。
- **系統底層驗證機制 (隱藏行為)：**
  - 當點擊「新增此筆交易」時，系統背景呼叫 Binance 公開 K 線 API (免費、無須金鑰)。
  - 系統抓取該日期該幣種的真實 `High` 與 `Low` 並算出系統平均值。
  - **防呆卡控：** 若使用者輸入的值與系統平均值誤差 > 5%，直接跳出報錯警告，拒絕新增該筆紀錄，並提示使用者重新檢查計算過程。
- UI 回饋：新增成功的紀錄會列於下方表格，並清楚標示「使用者自算換算匯率」與「驗證結果 (含系統抓取值)」。

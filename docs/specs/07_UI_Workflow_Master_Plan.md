# LumenTracker 全域工作流與 UI 規劃總覽 (Master UI Workflow)

依據專案決策者 (Deede) 的指示，本專案的完整操作動線 (UI 1 ~ UI 7) 必須嚴格遵循以下邏輯。任何開發與重構皆不得破壞此核心工作流。

## 系統進入點 (Entry Routing)
系統啟動時，會先檢查本地是否存在 `electron-store` 設定檔。
- **初次使用 (First Run)**：無設定檔，自動導向 **UI 1**。
- **後續使用 (Normal Start)**：已存在設定檔，自動導向 **UI 3 (登入)**。

---

## 階段一：系統初始化與資安綁定 (First-Time Setup)

### UI 1：歡迎與資安宣告 (Welcome Screen)
*   **功能**：宣告軟體之單機運行、BYOK (自備金鑰) 與零信任架構等核心理念。
*   **動線**：點擊「開始設定」 $\rightarrow$ **UI 2**。

### UI 2：使用者建檔 (Investigator Profile)
*   **功能**：建立首位調查員的實名檔案。
*   **欄位**：所屬單位、職稱、姓名、**聯絡電話 (作為系統唯一帳號)**、電子郵件 (選填)。
*   **動線**：點擊「下一步」 $\rightarrow$ **UI 3**。

### UI 3：密碼設定 / 系統登入 (Authentication)
*   **情境 A (初次設定)**：顯示「設定登入主密碼」。輸入兩次密碼以進行 PBKDF2 雜湊設定。
    *   **動線**：點擊「下一步」 $\rightarrow$ **UI 4**。
*   **情境 B (日常登入)**：顯示「系統已鎖定，歡迎回來 [職稱 姓名]」。輸入密碼解鎖本地資料庫與金鑰。
    *   **動線**：點擊「解鎖並進入系統」 $\rightarrow$ **UI 5**。

### UI 4：API Key 申請與綁定 (API Configuration)
*   **功能**：引導輸入 Etherscan (必備) 與 Tronscan (建議) 金鑰，並提供即時「測試連線」功能。
*   **動線**：點擊「完成設定並加密儲存」 $\rightarrow$ 背景寫入初始化稽核日誌 $\rightarrow$ **UI 5**。

---

## 階段二：案件管理樞紐 (Case Management)

### UI 5：案件管理樞紐 (Case Hub)
*   **功能**：系統的核心十字路口。顯示當前登入者身分。
*   **動線 A**：點擊「📄 建立新案件」 $\rightarrow$ 清空暫存變數 $\rightarrow$ **UI 5.5 (案件立案)**。
*   **動線 B**：點擊「📂 開啟歷史案件列表」 $\rightarrow$ 彈出 Modal 選擇內部資料庫案件 $\rightarrow$ 點擊載入 $\rightarrow$ 復原資料 $\rightarrow$ 直接跳至 **UI 6**。
*   **動線 C**：點擊「🛡️ 審核」 $\rightarrow$ 暫不開放。

### UI 5.5：鑑識調查中心 (Case Setup / 立案)
*(備註：這是確保證據不可否認性的強制閘道)*
*   **功能**：填寫「調查單位」、「調查員姓名」、「案件編號/名稱」、「初始已知涉案地址」。
*   **背景動作**：抓取當下時間戳記，生成「常駐證據鏈標頭」。
*   **動線**：點擊「立案並開始調查」 $\rightarrow$ **UI 6**。

---

## 階段三：鑑識工作區與分析

### UI 6：案件資料工作區 (Case Workspace)
*   **視覺特徵**：頂部帶有「證據鏈標頭 (黑底黃字)」。
*   **區塊一：案情摘要** (文字框)。
*   **區塊二：嫌疑錢包建檔**。具備「新增」按鈕與重複地址防呆機制。
*   **區塊三：交易紀錄 (被害人陳述)**。具備「新增」按鈕，需輸入 24 小時制時間，並有幣種下拉選單。背景會自動計算單筆匯率與調閱 API 獲取當日均價。
*   **按鈕動線**：
    *   「返回樞紐」 $\rightarrow$ **UI 5**。
    *   「💾 存入資料庫」 $\rightarrow$ 背景儲存至內部 `electron-store`。
    *   「▶ 進入畫布分析」 $\rightarrow$ **UI 7**。

### UI 7：鑑識分析與視覺化中心 (Analysis Dashboard)
*   **視覺特徵**：專業的「四方分割 (4-Panel)」佈局。
    *   **左側 (紅框)**：鑑識工具箱 (畫布縮放、過濾器、匯出快照)。
    *   **右上左 (黃框)**：錢包地址情報剖析 (啟用時間、Gas 來源)。
    *   **右上右 (綠框)**：雙頁籤設計。切換「鏈上真實交易」與「被害人筆錄對照」。
    *   **下方 (藍框)**：Cytoscape 涉案幣流關聯畫布 (Tracing Graph)。
*   **核心互動**：點擊畫布上的節點 $\rightarrow$ 右上方黃綠框瞬間更新 API 數據 $\rightarrow$ 由調查員「手動」挑選可疑交易加入畫布。

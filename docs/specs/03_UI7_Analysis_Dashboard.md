# UI 7: 鑑識分析與視覺化中心 (Analysis Dashboard)

## 核心設計理念 (DSR)
UI 7 拋棄了傳統黑箱鑑識軟體「一鍵生成」的模式，採用**「白箱/輔助思考 (White-box/Guided Analysis)」**的設計。
系統提供微觀數據與畫布工具，但「節點的展開與確認」必須由鑑識人員主觀判定並手動加入，以符合法庭證據的「解釋性 (Interpretability)」與「正當性 (Justifiability)」。

## 介面佈局規劃

### 1. 左側邊欄：功能列 (Sidebar)
作為鑑識工具箱，未來可擴充。目前核心功能如下：
*   **工具箱 (Tools)**：
    *   `切換選取模式 (Select/Pan)`
    *   `自動排版 (Auto-Layout)`：對使用者已加入的節點進行拓撲優化。
    *   `重置畫布中心 (Reset View)`
*   **圖層與過濾 (Layers & Filters)**：
    *   `隱藏/顯示 小額交易 (Dust/Spam)`
    *   `時間軸過濾 (Time-boxing slider)`
*   **匯出中心 (Export)**：
    *   `匯出鑑識報表 (PDF/CSV)`
    *   `匯出畫布快照 (PNG/SVG)`

### 2. 右側面板：微觀證據區 (Evidence Panel)
這是一個可收折 (Collapsible) 的面板，確保畫布空間最大化，但在需要核對資料時隨時可展開。
*   **子區塊 A：目標初勘與剖析 (Target Profiling)**
    *   延續 UI 6 自動分析的功能（啟用時間、Gas 來源、自動標籤）。
    *   當在畫布上點擊任何「節點」時，此區塊會即時更新為該節點的 API 剖析數據。
*   **子區塊 B：交易紀錄核對表 (Transaction Reconciliation)**
    *   **核心痛點解決**：解決「報案陳述時間」與「鏈上真實時間 (Block Time)」的落差。
    *   **UI 設計**：採用「並排對照 (Side-by-side)」或「顏色高亮」設計。左側顯示被害人在 UI 6 輸入的陳述，右側顯示系統透過 API 抓到的鏈上真實紀錄。
    *   若系統成功比對到 Hash，會顯示「✅ 已核對 (Verified)」，並以 UTC+8 標示真實區塊時間。

### 3. 中央主區塊：涉案幣流畫布 (Tracing Graph)
*   **呈現模式**：流程圖 (Flowchart) / 拓撲圖 (Topology Graph)。
*   **視覺元素**：
    *   **節點 (Nodes)**：代表錢包地址。以被害人錢包為中心出發。可透過圖示區分「個人錢包」、「交易所熱錢包 (中心化)」、「智能合約 (去中心化)」。
    *   **連線 (Edges)**：代表交易。每一條實體交易皆為一條獨立連線 (由 From 指向 To)，並在線上標註「金額、幣種、時間」。
*   **互動邏輯 (Crucial!)**：
    *   **非自動展開**：系統只會畫出被害人與第一層嫌疑地址的連線。
    *   **手動推進**：使用者必須點擊某個嫌疑節點，系統會在右側面板列出該節點的 API 交易歷史，由使用者判斷哪幾筆交易是「可疑洗錢流向」，然後**手動點擊「加入畫布」**。
    *   **由左至右延伸**：預設採用 Directed Acyclic Graph (DAG) 的由左至右分層佈局，清晰展現資金的「歸集」或「打散」過程。

## 實作技術建議
*   **圖表引擎**：建議採用 `Cytoscape.js` 或 `Vis.js`。兩者皆能完美處理大量節點的互動、拖曳，且支援以左至右的階層式 (Hierarchical) 排版。

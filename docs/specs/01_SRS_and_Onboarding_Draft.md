# 系統規格與初始教學流程草案 (SRS & Onboarding Draft v1.0)

## 1. 專案核心定位
- **軟體名稱**：（暫定，待 Deede 命名）
- **性質**：單機版區塊鏈幣流分析與數位鑑識教學軟體（繁體中文）。
- **目標受眾**：台灣司法機關調查員、執法人員、區塊鏈鑑識學員。
- **終極目標**：作為學術研究論文的核心實作，探討「無伺服器架構下的數位證據保全與法庭展演設計」。

## 2. 核心架構與資安原則 (CISO 視角)
- **全程操作同步驗證 (Real-time Operational Validation)**：
  - 系統將在操作者執行的每一步操作中，進行實時的同步驗證。這包括但不限於：數據輸入格式、API 返回值的預期性、特定操作的邏輯合規性。**特別是當系統偵測到潛在的跨鏈或混幣行為，而操作者可能未及發現時，系統將會主動發出視覺警示（如：醒目顏色、圖標或特殊標籤）、強制提示資訊（解釋異常、影響與建議追查策略），並詳細記錄操作者的回應與選擇。** 目標是從源頭確保操作的正確性與符合鑑識流程標準，同時提升調查員對複雜風險的感知。
- **零知識/無痕架構 (Zero-Knowledge/Trace-free Architecture)**：
  - 開發者（我們）不持有、不轉發使用者的任何查詢請求。
  - 完全由軟體於使用者本機端向公開區塊鏈節點（RPC/API）發起請求。
- **自備金鑰 (Bring Your Own Key, BYOK)**：
  - 軟體不內建共用 API Key，避免被封鎖或暴露整體調查單位的來源 IP 群。
  - 軟體具備教學模組，引導使用者自行註冊如 Etherscan, Alchemy 等免費帳號並填入 API Key。
- **本地加密儲存 (Local Encrypted Storage)**：
  - 所有操作足跡、查詢紀錄（包含 API 返回的原始 JSON 數據）均加密儲存於本機 SQLite 資料庫。
  - 確保偵查不公開原則，調查資料絕不上雲。
- **證據同一性與不可篡改性 (Evidential Integrity & Immutability)**：
  - 每一次操作或查詢的結果，均生成具備時間戳記的雜湊值（Hash）。
  - 最終產出的報告（PDF/DOC 等）附帶防篡改校驗碼，確保證據鏈在傳遞與法庭呈現時的完整性與可信度。

## 3. 核心功能模組

### 3.1. 基礎設定與教學模組 (Onboarding & API Setup)
- 軟體首次啟動時的引導精靈。
- 圖文並茂地教導使用者「什麼是 API Key」、「為什麼我們不提供共用金鑰（資安與隱私考量）」、「如何去 Etherscan 申請並貼上」。
- 驗證金鑰有效性後，才允許進入分析介面。

### 3.2. 互動式資金追蹤介面 (Interactive Tracing Map)
- 輸入嫌疑地址作為起點。
- 軟體不自動遞迴展開所有交易，而是列出該地址的交易列表，讓使用者「手動點擊」展開下一層。
- 具備過濾小額交易（找零）、標記可疑地址、添加個人備註的功能。
- **操作同步驗證與足跡記錄**：使用者的每一次點擊、過濾條件、筆記、數據輸入等操作，都會被背景的「足跡引擎」即時記錄下來。同時，系統會對每一步關鍵操作進行**同步驗證**，例如：
  - **數據格式驗證**：確保輸入的地址或交易哈希符合區塊鏈標準格式。
  - **API 返回值驗證**：檢查從區塊鏈 API 獲取的數據是否完整、正確，並符合預期格式。
  - **操作邏輯驗證**：確保追蹤流程符合預設的鑑識邏輯（例如，在追蹤跨鏈交易時提示潛在風險）。
  - **完整性檢查**：對關鍵數據點進行哈希驗證，確保在操作過程中數據未被意外篡改。
  **特別是當系統偵測到潛在的跨鏈或混幣行為，而操作者可能未及發現時，將會：**
  1.  **主動視覺化警示**：在介面中以醒目的顏色、圖標或特殊標籤（如「潛在跨鏈點」、「可疑混幣交易」）高亮顯示。相關交易資訊旁出現警告符號（⚠️）。
  2.  **強制確認與資訊提示**：彈出強制性提示視窗，解釋異常類型、對追查的影響、建議的追查策略。調查員必須明確點擊「確認」或「忽略風險但記錄」後才能繼續。
  3.  **風險評分自動提升**：相關地址或交易在「風險評分模型」中自動賦予更高的風險權重和評分。
  4.  **詳盡日誌記錄**：足跡引擎將記錄所有警示細節、操作員回應與系統自動調整的風險評分。
  這些驗證結果也將一併記錄在足跡中，以強化證據的不可否認性。

### 3.3. 法庭展演模式 (Courtroom Replay / Playback Mode)
- **封裝案件包 (Case File)**：將調查過程中的所有足跡與當時的區塊鏈原始數據打包成單一加密檔案。
- **離線播放**：在法庭上斷網環境中載入案件包。
- **迷宮破解展示**：如同簡報或解謎遊戲般，一步步（Step-by-step）重現資金流向的思考邏輯，展示畫面與當時調查時完全一致。**這也包括重現所有系統發出的「警示彈窗」和「風險提示」**，以及調查員當時選擇「確認」或「忽略」的互動，以增強證據的透明度和說服力。

### 3.4. 鑑識報告生成器 (Forensic Report Generator)
- 根據「足跡引擎」的紀錄，自動產生敘述性的偵查報告。
- 支援格式：`.doc`, `.odt`, `.csv`, `.pdf`, `.png`。
- 報告內容將嚴格遵循 **[鑑識報告章節結構設計](02_Forensic_Report_Structure.md)** 中定義的各項細節，確保報告的全面性與符合司法要求。

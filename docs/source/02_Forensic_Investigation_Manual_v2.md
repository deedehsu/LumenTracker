## 第一章：數位證物進件與錢包「解剖」
**偵查思維**：拿到地址的第一秒，不是去搜，而是先分類。

### 一、 地址格式與鏈別辨識
1. **比特幣 (BTC)**：
   - 數字 `1` 開頭，共 34 字元，區分大、小寫。
   - 數字 `3` 開頭，共 34 字元，區分大、小寫。
   - `bc1q` 開頭，共 42 字元，不區分大、小寫。
   - `bc1p` 開頭，共 62 字元，不區分大、小寫。
2. **波場鏈 (TRON) / TRC-20**：`T` 開頭，共 34 字元，區分大、小寫。常見於 USDT 詐騙，交易速度快、成本低。
3. **以太鏈 (ETH) / ERC-20**：`0x` 開頭，共 42 字元，區分大、小寫。需注意包含 ETH, BSC (幣安鏈), Polygon 等多鏈相容。

### 二、 公開帳本查詢 (以 TRC-20 為例)
- **工具**：TRONSCAN (https://tronscan.org)
- **查詢資料**：
  - **帳戶概況**：啟用時間、交易次數。
  - **餘額**：TRX 及 USDT。
  - **權限設定 (Permission)**：檢查控制權是否移轉（100% 轉走即為傀儡錢包）。
  - **交易紀錄**：排除小額粉塵攻擊，專注於 From, To, Amount, Time。
  - **資源管理**：能量與頻寬（質押或租賃）。

### 三、 錢包性質判定與辨識
1. **交易所入金地址 (Deposit)**：資金匯入後，隨即「歸集」至大水庫。
2. **交易所水庫 (Hot Wallet)**：交易量極大，通常帶有標籤 (如 "Binance-Hot")。
3. **手續費來源 (Gas Provider)**：觀察啟動資金（第一筆 TRX）是誰給的，若多個地址的 TRX 來自同一地址，即構成 **集群 (Cluster)**。
4. **資源租賃行為**：若 TRX 交易有「代理資源」或極小額（如 1.1 TRX）轉入，代表嫌犯透過第三方租用能源規避追蹤。

## 第二章：資產剖析與「生存資源」追查
**偵查思維**：沒有 Gas，錢包就是死的。

### 📌 常見鏈種手續費 (Gas) 對照表
在追查前，必須先釐清該條區塊鏈是消耗何種「原生代幣」作為手續費：
- **比特幣 (BTC)** ➔ `BTC`
- **以太鏈 (Ethereum)** ➔ `ETH`
- **波場鏈 (TRON)** ➔ `TRX`
- **幣安智能鏈 (BSC)** ➔ `BNB`
- **Polygon** ➔ `POL` (原 MATIC)
- **TON** ➔ `Toncoin`

1. **啟動資金 (Genesis Transaction)**：找出第一筆轉入的原生代幣（對應上述手續費幣別），若來自個人錢包，該錢包即為「資源供應源」。
2. **手續費來源 (Gas Provider)**：
   - 單一供應源：多個涉案錢包由同一地址付手續費 -> 同一犯罪組織。
   - 加油站模式：專門的「加油錢包」定額發送 TRX/ETH。
3. **能源與頻寬 (Tron 專屬)**：觀察 Internal Transactions 中是否有能源轉入（租借），支付路徑常暴露嫌犯真實錢包。

## 第三章：錢包集群 (Cluster) 與行為建模
**偵查思維**：證明「這群錢包」都是「同一個人」控管。

1. **共同輸入權屬 (CIO)**：多個地址同時作為交易 Input。
2. **找零地址 (Change Address)**：交易後餘額匯入的新地址，後續有歸集行為。
3. **手續費指紋**：嫌犯發送 Gas 的頻率、金額具備一致性。

---

## 第四章：進階交易邏輯分析 (換幣、跨鏈、授權)
**偵查思維**：當線索斷掉時，嫌犯可能正在「切換維度」。

1. **換幣 (Swap / DEX)**：USDT 換成 ETH，需解析 Input Data 找流向。
2. **跨鏈 (Bridge)**：從 Tron 跳到 Ethereum，需利用跨鏈瀏覽器反推 TxID。
3. **授權詐騙 (Approve)**：檢查 Approve 紀錄找出惡意合約。

---

## 第五章：反偵查手段辨識 (混幣與剝皮)
1. **剝皮鏈 (Peeling Chain)**：大額資金分散小額轉帳，對策為專注「大額餘數」去向。
2. **混幣器 (Mixers)**：如 Tornado Cash，需觀察進出時間與金額統計關聯性。

---

## 第六章：偵查終點——交易所鎖定與調閱
**偵查思維**：不僅調閱「贓款出口」，更要調閱「資源入口」。

1. **雙向調閱策略**：
   - **出口調閱**：資金最終進入交易所的水庫前，最後一次停留的「入金地址 (Deposit Address)」。
   - **入口調閱 (關鍵突破)**：供應手續費的 TRX/ETH 最初是從哪個交易所提領出來的。
2. **調閱實務**：需附上 TxID、精確時間 (UTC+8)、From/To 地址。

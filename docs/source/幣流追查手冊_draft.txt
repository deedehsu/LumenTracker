第一章：數位證物進件與錢包「剖析」
偵查思維： 拿到地址的第一秒，不是去搜，而是先分類。
一、 地址格式與鏈別辨識
	•	TRC-20 (Tron)： T 開頭34字元區分大、小寫。常見於 USDT 詐騙，交易速度快、成本低。
(一)、 如何使用公開帳本查詢TRC-20 (Tron)？
	•	在 Tron (波場) 鏈上，最權威且最直接的查詢工具是官方的區塊鏈瀏覽器： TRONSCAN (https://tronscan.org) 。
	•	操作步驟：進入首頁搜尋框→輸入以 「T」開頭 的錢包地址、64 位元的交易哈希值 (TxID或稱HASH) 或合約地址→按下 Enter。
	•	
(二)、 公開帳本能查出哪些資料？
	•	帳戶概況 (Account Overview)：啟用時間、發起交易方次數、接受交易方次數、交易總額等資訊。
	•	餘額 (Balance)： 該地址目前持有的 TRX 及 USDT (TRC-20) 數量 。
	•	權限設定 (Permission)： 檢查該帳號的控制權是否已被轉移（若控制權 100% 轉走，則為傀儡錢包） 。
	•	交易紀錄 (Transactions)：應先排除小額交易及粉塵攻擊
	•	轉帳明細： 包含每一筆資產的 From (來源)、To (去向)、Amount (金額) 及 Time (精確時間) 。
	•	手續費消耗： 該筆交易消耗了多少 TRX 或是多少能量 (Energy) 。	
	•	資源管理 (Resources)：
	•	能量與頻寬： 該地址是否具備質押獲取的資源，或是正在使用「能量租賃」服務 。
	•	
(三)、 所查資料如何辨識與判斷？
	•	辨識「錢包角色」：
(1)、交易所入金地址 (Deposit)： 若資金匯入後，隨即發生「歸集」動作進入一個交易量極大的地址，代表極可能為交易所的託管錢包 。
(2)、交易所水庫 (Hot Wallet)： 交易筆數通常高達數十萬筆，且標籤會標註如 "Binance-Hot" 。
	•	辨識「手續費來源」 (關鍵突破點)：
(1)、觀察該地址的第一筆 TRX 或是轉帳前入帳的整數顆 TRX (或當前協議手續費) 是誰給的 (14、30、100等數量)。
(2)、若多個嫌疑地址的 TRX 都來自同一個地址，這就是一個 Cluster (集群)，代表他們屬於同一個犯罪組織 。
	•	辨識「資源租賃行為」：如果在 TRX 交易紀錄中看到「代理資源」或轉入金額如 1.1 TRX ，代表嫌犯正透過第三方平台租用能源以規避 TRX 消耗，這通常是組織化犯罪規避追蹤的手法 。
	•	💡 Lumen 導師的小提醒：
	•	辨識資料時，請務必養成**「時區換算」**的習慣。公開帳本多使用 UTC+0 時間，產出報告時需換算為台灣時間 (UTC+8)，才能與被害人的筆錄時間對齊 。
	•	您是否需要我針對「如何看懂 Tronscan 的 Permission (權限) 頁面」做更詳細的解說？這對於辨識是否為受控傀儡錢包非常重要。
	•	ERC-20 (Ethereum/EVM)： 0x 開頭。需注意包含 ETH, BSC (幣安鏈), Polygon 等多鏈相容。
	•	BTC (Bitcoin)： 1, 3, 或 bc1 開頭。
	•	針對 ERC-20 (Ethereum/EVM) 體系的公開帳本查詢與資料辨識，以下為您整理的實戰指南：
	•	一、 如何使用公開帳本查詢？
	•	由於 ERC-20 標準在多條鏈（如 Ethereum, BSC, Polygon）上通用，偵查員需根據「資產所在地」選擇對應的瀏覽器：
	•	主流工具：
	•	Ethereum (以太坊)： Etherscan (https://etherscan.io) 。
	•	BNB Smart Chain (幣安智能鏈)： BscScan (https://bscscan.com) 。
	•	全鏈通用 (推薦)： OKLink (https://www.oklink.com)，支援多鏈切換，適合追蹤跨鏈資金 。
	•	操作步驟： 在搜尋框輸入以 「0x」開頭 的錢包地址或 64 位元的交易哈希值 (TxID) 即可 。
	•	
	•	二、 公開帳本能查出哪些資料？
	•	查詢後，系統會展示該地址的「鏈上身分證」，包含以下核心偵查數據：
	•	資產總覽 (Balance & Tokens)： 顯示該地址擁有的原生代幣（如 ETH/BNB）以及所有 ERC-20 代幣（如 USDT, USDC）的餘額與匯率 。
	•	交易列表 (Transaction History)：
	•	Transactions： 包含主網原生代幣轉帳及合約交互紀錄 。
	•	Internal Transactions： 合約內部的資金流轉（如 Uniswap 換幣後的內部拆分），這是偵查「換幣」的關鍵 。
	•	ERC-20 Token Transfers： 專門記錄 USDT 等代幣的轉帳軌跡 。
	•	合約交互訊息 (Contract Interaction)： 若與合約互動，會顯示函數名稱（如 Swap 代表換幣，Approve 代表授權） 。
	•	
	•	三、 所查資料如何辨識與判斷？
	•	拿到數據後，請依照以下邏輯進行「行為分析」：
	•	1. 辨識「交易所與實體」：
	•	標籤辨識 (Labeling)： 留意地址旁是否出現如 Binance Exchange 或 FTX 的標籤 。
	•	入金行為： 若資金進入後隨即轉入一個標記為 Hot Wallet 的大錢包，代表該 0x 地址為用戶在交易所的入金地址 。
	•	2. 辨識「授權風險 (Approve)」：
	•	若在交易紀錄中看到大量 Approve 動作，代表該錢包已授權給某個合約。若合約名稱不明，極可能是「授權盜取」類型的詐騙 。
	•	3. 辨識「換幣 (Swap)」：
	•	檢查 Input Data。若被害人轉入 USDT，但該地址後續出現 OneInch 或 Uniswap 字樣，代表資產已被換成其他幣種（如 ETH），偵查員需切換至 Internal Transactions 繼續追蹤 。
	•	4. 辨識「跨鏈行為 (Bridge)」：
	•	若資金轉入如 SWFT 或 Multichain 的合約地址，鏈上追蹤會在此中斷。此時需使用「跨鏈瀏覽器」，利用該筆 TxID 反推其在目的鏈（如從 ETH 跳到 Tron）的新 TxID 。
	•	💡 Lumen 導師的小提醒：
	•	ERC-20 地址在不同鏈上可能是同一個字串，但裡面的錢卻是獨立的。在製作筆錄時，務必請被害人指明是在「哪條鏈」操作，否則會在正確的地址裡看到錯誤的餘額。
	•	下一個步驟，您是否希望我針對「如何解析 Input Data 來識別換幣細節」進行更深入的圖解教學？
	•	
2. 錢包性質判定
	•	EOA (外部帳戶)： 個人控管的錢包（如 Trust Wallet）。有私鑰才有控制權。
	•	Contract (合約地址)： 具備邏輯程式碼。若資金匯入此處，可能是進入了 Swap (換幣)、Bridge (跨鏈) 或 Scam (詐騙) 合約。
	•	託管錢包 (CEX)： 交易所分配給用戶的「入金地址」。
	•	權限檢查 (Permission Update)： 檢查錢包的 Owner 或 Active 權限是否被修改。若權限移轉給第三方，則該錢包的資源（Energy/Bandwidth）可能由外部控制。

第二章：資產剖析與「生存資源」追查
偵查思維：沒有 Gas，錢包就是死的。
1. 啟動資金 (Starting Funds / Genesis Transaction)
	•	分析內容： 找出該錢包在區塊鏈上的「第一筆交易」。
	•	偵查點： * 誰轉入第一筆原生代幣（TRX, ETH, BNB）？
	•	如果第一筆錢來自交易所，記錄該交易所名稱與時間。
	•	如果來自另一個個人錢包，該錢包即為**「資源供應源」**。
2. 手續費來源 (Gas Provider / Refueling)
	•	分析內容： 錢包在轉出贓款前，必定有手續費入帳。
	•	偵查點： * 單一供應源： 多個涉案錢包的手續費都由同一個地址支付 $\rightarrow$ 直接判定為同一集群 (Cluster)。
	•	加油站模式： 嫌犯設立專門的「加油錢包」，定額發送小額 TRX/ETH 給車手。
3. Tron 專屬資源：能源與頻寬 (Energy & Bandwidth)
	•	背景： 在 Tron 鏈上，轉帳 USDT 需消耗大量「能源」。嫌犯若不想直接支付 TRX，會採取「資源管理」手段。
	•	分析內容： * 質押 (Staking)： 錢包是否質押了 TRX 以獲得免費能源？
	•	能源租賃 (Energy Leasing)： 錢包是否透過合約（如：能源平台）租用資源？
	•	偵查點： * 檢查交易紀錄中的 Internal Transactions，看是否有「能源轉入」。
	•	識別能源供應商。雖然可能是合法平台，但租借資源的支付路徑往往會暴露嫌犯的真實錢包。

第三章：錢包集群 (Cluster) 與行為建模
	•	1. 共同輸入權屬 (Common Input Ownership, CIO)
	•	原理： 多個地址同時作為一筆交易的 Input，證明其私鑰由同一實體控制。
	•	偵查點： 在分析工具中若看到多點匯流成一條線，這就是一個 Cluster。
	•	2. 找零地址 (Change Address)
	•	辨識： 交易後剩餘的金額匯入的一個全新、且後續有資金歸集行為的地址。
	•	3. 手續費來源 (Gas Provider)
	•	關鍵突破口： 追蹤是誰提供第一筆 TRX 或 ETH 給涉案錢包。通常嫌犯會用同一個錢包為數十個傀儡錢包「發薪水」。
	•	手續費指紋： 嫌犯即便頻繁換地址，但其 Gas 的發送頻率、金額（例如每次都發送剛好 30 TRX）通常具備一致性，可作為行為特徵比對。

第一章：數位證物進件與錢包「解剖」
偵查思維： 拿到地址的第一秒，不是去搜，而是先分類。
1. 地址格式與鏈別辨識
	•	TRC-20 (Tron)： T 開頭。常見於 USDT 詐騙，交易速度快、成本低。
	•	ERC-20 (Ethereum/EVM)： 0x 開頭。需注意包含 ETH, BSC (幣安鏈), Polygon 等多鏈相容。
	•	BTC (Bitcoin)： 1, 3, 或 bc1 開頭。
2. 錢包性質判定
	•	EOA (外部帳戶)： 個人控管的錢包（如 Trust Wallet）。有私鑰才有控制權。
	•	Contract (合約地址)： 具備邏輯程式碼。若資金匯入此處，可能是進入了 Swap (換幣)、Bridge (跨鏈) 或 Scam (詐騙) 合約。

第二章：資產剖析與「水位」監控
偵查思維： 餘額是「當下」，流量是「歷史」。
1. 水位查詢 (Balance & Flow)
	•	即時餘額： 決定是否具備「緊急攔阻/凍結」的價值。
	•	總體流量 (Total Received)： 判斷該錢包的「量體」。流量極大但餘額趨近於零的地址，通常是嫌犯的「轉運水庫」。
2. 匯率與價值換算
	•	換算標準： 必須記錄被害當時的法幣匯率 (USD/TWD)。
	•	異常代幣： 若收到大量名稱奇怪的代幣，需確認是否為無價值的空氣幣。

第三章：錢包集群 (Cluster) 與行為建模
偵查思維： 證明「這群錢包」都是「同一個人」控管。
1. 共同輸入權屬 (Common Input Ownership, CIO)
	•	原理： 多個地址同時作為一筆交易的 Input，證明其私鑰由同一實體控制。
	•	偵查點： 在分析工具中若看到多點匯流成一條線，這就是一個 Cluster。
2. 找零地址 (Change Address)
	•	辨識： 交易後剩餘的金額匯入的一個全新、且後續有資金歸集行為的地址。
3. 手續費來源 (Gas Provider)
	•	關鍵突破口： 追蹤是誰提供第一筆 TRX 或 ETH 給涉案錢包。通常嫌犯會用同一個錢包為數十個傀儡錢包「發薪水」。

第四章：進階交易邏輯分析 (換幣、跨鏈、授權)
偵查思維： 當線索斷掉時，嫌犯可能正在「切換維度」。
1. 換幣 (Swap / DEX)
	•	行為： 在 Uniswap 或 PancakeSwap 將 USDT 換成 ETH 或其他高波動幣種。
	•	追蹤： 需解析 Input Data，找出換幣後的目標資產與流向。
2. 跨鏈 (Bridge)
	•	行為： 資金從 Tron 跳到 Ethereum。
	•	反追蹤： 利用跨鏈瀏覽器，輸入 Receive Hash 反推 Deposit Hash，找回來源鏈的始作俑者。
3. 授權詐騙 (Approve / Allowance)
	•	行為： 被害人錢包未主動轉帳，但錢被搬空。
	•	偵查點： 檢查 Approve 紀錄，找出被授權的惡意合約地址。


第五章：反偵查手段辨識 (混幣與剝皮)
1. 剝皮鏈 (Peeling Chain)
	•	特徵： 大額資金分散成無數小額轉帳，每一手剝掉一點點。
	•	對策： 忽略細枝末節，專注於「大額餘數」的最終去向。
2. 混幣器 (Mixers)
	•	特徵： 資金進入 Tornado Cash 等平台。
	•	偵查點： 觀察進出時間與金額的統計關聯性，標記該地址為「高度洗錢風險」。

第六章：偵查終點——交易所鎖定與調閱
偵查思維：不僅調閱「贓款出口」，更要調閱「資源入口」。
1. 雙向調閱策略
	•	出口調閱： 資金最終進入交易所的 $TxID$。
	•	入口調閱 (關鍵)： 供應這群錢包手續費的原生代幣（TRX/ETH），最初是從哪個交易所帳號提領出來的？
	•	1. 識別交易所 (CEX Identification)
	•	入金地址 (Deposit Address)： 每個用戶專屬，是調閱人別資料的唯一憑據。
	•	交易所水庫 (Hot Wallet)： 資金一旦進入水庫，鏈上追蹤即終止。
	•	2. 調閱實務與報告產出
	•	報告要素： $TxID$ (64位元雜湊值)、精確時間 (UTC+8)、From/To 地址。
	•	調閱窗口： * 國內：交易所法規窗口。
	•	國外：幣安 (Kodex 系統)、Tether (凍結請求信箱)、OKX 等。
	•	
	•	Lumen 提示： 嫌犯常在 A 交易所入金（洗錢），但在 B 交易所提領手續費。兩者皆具備人別辨識價值。
